# Resume Screening Pro

A Flask-based resume screening application with React frontend for analyzing and matching resumes against job descriptions.

## Split Deployment: Backend on Render + Frontend on Vercel

This project is configured for split deployment with backend on Render and frontend on Vercel.

### Prerequisites

1. Push your code to a GitHub repository
2. Create accounts at [Render](https://render.com) and [Vercel](https://vercel.com)
3. Have your GROQ API key ready

### Backend Deployment (Render)

**Environment Variables:**
- `GROQ_API_KEY`: Your Groq API key for AI processing
- `SECRET_KEY`: Auto-generated by Render
- `DATABASE_URL`: Auto-configured by Render (PostgreSQL)

**Steps:**
1. In Render dashboard, click "New +" â†’ "Web Service"
2. Connect your GitHub repository
3. Render will auto-detect the `render.yaml` configuration
4. Add your `GROQ_API_KEY` as environment variable
5. Deploy - Backend will be available at: `https://resume-screening-api.onrender.com`

### Frontend Deployment (Vercel)

**Environment Variables:**
- `VITE_API_URL`: Set to your Render backend URL (`https://resume-screening-api.onrender.com`)

**Steps:**
1. In Vercel dashboard, click "New Project"
2. Import your GitHub repository
3. Set root directory to `frontend`
4. Add `VITE_API_URL` environment variable
5. Deploy - Frontend will be available at your Vercel URL

### Important Configuration Updates

**After deployment, update CORS origins in `backend/app.py`:**
Replace `https://your-vercel-app.vercel.app` with your actual Vercel URL.

**Update frontend environment:**
In `frontend/vercel.json`, ensure `VITE_API_URL` points to your Render backend.

### Local Development

**Backend:**
```bash
cd backend
pip install -r requirements.txt
python app.py
```

**Frontend:**
```bash
cd frontend
npm install
npm run dev
```

### Services Configuration

- **Backend (Render)**: Flask + Gunicorn + PostgreSQL
- **Frontend (Vercel)**: React static site
- **Database**: PostgreSQL provided by Render
- **Health Check**: `/health` endpoint on backend

### Notes

- Backend automatically switches between SQLite (local) and PostgreSQL (production)
- CORS is configured for local development, Render, and Vercel URLs
- Frontend build artifacts are excluded from git via .gitignore
- Remember to update the CORS origins after getting your Vercel URL

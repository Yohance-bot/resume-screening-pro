# Resume Screening Pro

A Flask-based resume screening application with React frontend for analyzing and matching resumes against job descriptions.

## Deployment on Render

This project is configured for easy deployment on Render using the `render.yaml` configuration file.

### Prerequisites

1. Push your code to a GitHub repository
2. Create a Render account at [render.com](https://render.com)
3. Have your GROQ API key ready

### Environment Variables

Set these environment variables in your Render dashboard:

**Backend Service:**
- `GROQ_API_KEY`: Your Groq API key for AI processing
- `SECRET_KEY`: Auto-generated by Render
- `DATABASE_URL`: Auto-configured by Render (PostgreSQL)

**Frontend Service:**
- `VITE_API_URL`: Automatically set to your backend URL

### Deployment Steps

1. **Connect Repository**
   - In Render dashboard, click "New +" â†’ "Web Service"
   - Connect your GitHub repository
   - Render will automatically detect the `render.yaml` configuration

2. **Configure Services**
   - The `render.yaml` file defines three services:
     - `resume-screening-api`: Flask backend
     - `resume-screening-frontend`: React frontend
     - `resume-screening-db`: PostgreSQL database

3. **Set Environment Variables**
   - Add your `GROQ_API_KEY` to the backend service
   - Other variables are auto-configured

4. **Deploy**
   - Render will automatically build and deploy your application
   - The backend will be available at: `https://resume-screening-api.onrender.com`
   - The frontend will be available at: `https://resume-screening-frontend.onrender.com`

### Local Development

**Backend:**
```bash
cd backend
pip install -r requirements.txt
python app.py
```

**Frontend:**
```bash
cd frontend
npm install
npm run dev
```

### Services Configuration

- **Backend**: Runs on Gunicorn WSGI server
- **Frontend**: Static site served by Render
- **Database**: PostgreSQL provided by Render
- **Health Check**: `/health` endpoint on backend

### Notes

- The application automatically switches between SQLite (local) and PostgreSQL (production)
- CORS is configured for both local development and production URLs
- Build artifacts are excluded from git via .gitignore
